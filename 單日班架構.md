# CLI、Git 與 Agentic AI 實作 — 從命令列到 AI 協作

## 基本資訊

- **時長：** 8 小時（09:00 – 17:00）
- **對象：** 一般民眾，無技術背景，想學 Agentic AI 工具
- **目標：** 從零學會命令列操作，掌握 Git 版本控制，能用 Agentic AI 協作完成任務
- **定位：** Agentic AI 入門 — CLI 是門票，Git 是基礎，AI 是加速器
- **人數：** TBD
- **場地：** TBD（學員自帶筆電，需有網路）
- **平台：** Mac 和 Windows 同時教，投影片雙色標示

## 課程設計理念

### 三層能力架構

| 層次 | 能力 | 說明 |
|------|------|------|
| CLI 操作 | 命令列基礎 | 不怕黑畫面，能操作檔案和目錄 |
| Git 版本控制 | 版本追蹤與協作 | 知道怎麼存版本、推到 GitHub |
| Agentic AI 協作 | 用自然語言指揮 AI | AI 在命令列幫你做事，你負責審查 |

- 開場帶出三層地圖：CLI → Git → Agentic AI
- 收尾回顧三層能力，連結到日常使用

### 設計原則

- **手把手實作為主**：每個指令都自己打一遍
- **Mac 和 Windows 並行**：橘色 = Mac、藍色 = Windows
- **不怕犯錯**：命令列錯誤不會弄壞電腦
- **漸進難度**：上午 CLI → 下午 Git → 最後 AI 整合
- **每段都有實作練習**，不是只看講師 demo

## 實作設計（15:15-16:00）

### 主軸：CLI + Git + AI 一條龍

| 步驟 | 做什麼 | 涉及能力 |
|------|--------|---------|
| 1 | mkdir + cd 建專案資料夾 | CLI |
| 2 | git init 初始化版本控制 | Git |
| 3 | claude 啟動 AI 助手 | Agentic AI |
| 4 | 自然語言描述目標，AI 實作 | Agentic AI |
| 5 | git diff 看改了什麼 → git add → commit | Git + 審查 |
| 6 | git push 推到 GitHub | Git |

### 每步節奏

```
講師 demo（3 min）→ 學員跟做（5 min）→ 確認完成（2 min）
```

## 常見地雷

| 地雷 | 說明 | 解法 |
|------|------|------|
| rm 刪了就沒了 | 沒有資源回收桶 | 刪之前用 ls 確認，或加 -i |
| 忘記 commit 就 push | 三步缺一不可 | status → add → commit → push |
| AI 幻覺 | AI 可能生成不存在的指令 | 看不懂就查，不要盲抄 |
| 環境變數外洩 | API Key 不要寫進程式碼 | 用 .env 檔案，.gitignore 排除 |

## 課後跟進

### 帶走的東西

- [ ] CLI 指令小抄（雙面）
- [ ] Git 指令小抄（雙面）

### 課後 3 天內群組互動

- 丟小挑戰：「用 git 建一個 repo，push 一個 commit，截圖到群組」
- 保持熱度，收集真實使用案例
- 有互動的群組才推得動進階班

### 進階班銜接

單日班 = CLI + Git + Agentic AI 入門
進階班 = AI 工作流整合（Claude Code 深入、MCP/Skills、n8n 自動化、CI/CD）

課程中埋的 hook：
- 開場三層地圖：「今天把三層地基打好」
- AI 實作段落：「進階班會教你讓 AI 自己跑整個工作流」
- 收尾：「三層能力你都有了，下一步是把它們串成自動化流程」

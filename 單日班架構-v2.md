# CLI 與 AI HTML 實作 — 單日班架構 v2

> 2026-06-17 更新：課程改為初學者體驗版。主線從「CLI + Git + Agentic AI 工程工作流」改成「用 AI 做出第一個本機 HTML 作品，順手認識 CLI」。Git 改為 demo / 加分，不再是必達成果；不教公開網址、不教 Vercel / Supabase / GitHub Pages。

## 設計變更摘要

| 項目 | 原設計 | 新設計 | 變更理由 |
|------|--------|--------|---------|
| 課程定位 | Agentic AI 入門，CLI 是門票、Git 是橋樑、AI 是目的地 | AI 做出第一個 HTML 作品，CLI 是操作入口 | 學員多半只熟 ChatGPT 對話，先看到成果再補工具 |
| 畢業成果 | CLI + Git + AI 一條龍，commit / push | 本機瀏覽器打開自己的 `index.html` | 降低失敗點，讓每個人感覺有做出東西 |
| Git | Unit 3 主線，學員跟做 add / commit / push | 講師 demo + 加分練習，只做本機 commit，不 push | 版控不是這堂最重要的學習目標 |
| GitHub / 部署 | 學員推上 GitHub | 不教公開網址 | 避免帳號、權限、網路、部署平台拖垮初學者 |
| CLI 深度 | pwd / cd / ls / mkdir / touch / cp / mv / rm / grep / pipe / env | 只保留做 HTML 需要的生存指令 | 初學者先會用，不先追求完整指令表 |
| AI 段落 | 下午第二節才大量實作 | 午前就做出第一個 HTML，下午用 AI 改版 | 讓學員早點看到「我做到了」 |
| 成功標準 | 走完現代開發工作流 | 看到自己的網頁、能請 AI 修改、能說出檔案在哪 | 對非工程背景更可靠 |

---

## 基本資訊

- **時長：** 8 小時（09:00 – 17:00，含 10min 緩衝）
- **對象：** 一般民眾，無技術背景，頂多熟悉 ChatGPT 對話
- **定位：** 初學者 AI 實作體驗課 — 用 AI 做出第一個本機 HTML 作品
- **必達成果：** 每位學員都有一個本機可開啟的 `index.html`
- **不做事項：** 不公開網址、不教 Vercel / Supabase / GitHub Pages、不要求學員 push
- **平台：** Mac 和 Windows 同時教，投影片雙色標示

---

## 課程設計理念

### 三層能力架構

| 層次 | 能力 | 定位 | 學員帶走什麼 |
|------|------|------|-------------|
| AI 對話 | 把想法說清楚 | 熟悉入口 | 會把「我要什麼網頁」講成 AI 能執行的指令 |
| CLI 操作 | 找檔案、進資料夾、開檔案 | 操作入口 | 不怕黑畫面，知道作品在哪裡 |
| HTML 作品 | 本機網頁產出 | 可見成果 | 打開瀏覽器看到自己的頁面，能再請 AI 修改 |

### 設計原則

- **成果優先**：先讓學員看到自己的網頁，再補工具概念。
- **少教名詞**：每個指令都要服務 `index.html` 成果，不為了完整而教。
- **Git 降級**：Git 只作為「可以保存版本」的概念 demo，不是主線。
- **不部署**：公開網址、Vercel、Supabase、GitHub Pages 全部移到進階課。
- **每段都有可見回饋**：CLI 後看得到檔案，AI 後看得到網頁，修改後看得到差異。

## 課程流程

### UNIT 1：開場 + AI 能做什麼 + CLI 生存感（09:00 – 10:30，90 min）

| 時間 | 主題 | 形式 | 細節 |
|------|------|------|------|
| 09:00-09:08 | 歡迎 + 課程重新定位 | 講 | 今天不是工程師訓練，是做出第一個本機 HTML |
| 09:08-09:20 | 自我介紹 + 作品展示 | 講 | 用真實作品建立可信度，但不要炫技 |
| 09:20-09:35 | ChatGPT 對話 → AI 做事 | 講 + 互動 | 從學員熟悉的 ChatGPT 切入，展示「一句話變成網頁」 |
| 09:35-09:45 | 黑畫面恐懼 | 互動 | 問感受，建立今天只需要會幾個指令 |
| 09:45-10:00 | 打開終端機 + 看自己在哪 | 跟做 | Terminal / PowerShell、`pwd`、`ls` |
| 10:00-10:15 | 進資料夾 + 建資料夾 | 跟做 | `cd Desktop`、`mkdir my-page`、`cd my-page` |
| 10:15-10:30 | 建第一個檔案 | 跟做 | `touch index.html` / `New-Item index.html`，確認 `ls` 看得到 |

**過關標準：** 每個人終端機能進到 `my-page`，且 `ls` 看得到 `index.html`。

**可刪減：** 自我介紹縮短；AI 展示從 4 張卡縮成 1 張「一句話變網頁」。

---

### 休息 15 min（10:30 – 10:45）

---

### UNIT 2：第一個本機 HTML（10:45 – 12:10，85 min）

| 時間 | 主題 | 形式 | 細節 |
|------|------|------|------|
| 10:45-11:00 | HTML 是什麼 | 講 | 不教語法大全，只講「網頁就是一個文字檔」 |
| 11:00-11:20 | 用 AI 產生第一版 HTML | 跟做 | 請 AI 產生簡單個人介紹頁，貼到 `index.html` |
| 11:20-11:35 | 本機瀏覽器打開 | 跟做 | Mac: `open index.html`；Windows: `start index.html` |
| 11:35-11:55 | 請 AI 改第一版 | 跟做 | 加標題、顏色、三個卡片、聯絡方式 |
| 11:55-12:10 | 小成果確認 | 互動 | 每人瀏覽器看到自己的頁面；卡住者用講師範本救援 |

**過關標準：** 本機瀏覽器能打開 `index.html`，畫面不是空白。

**救援方案：** 準備 `starter-index.html`，卡住者直接貼上再修改名字。

---

### 午餐 55 min（12:10 – 13:05）

---

### UNIT 3：AI 改版練習 — 從「能看」到「像作品」（13:05 – 14:35，90 min）

| 時間 | 主題 | 形式 | 細節 |
|------|------|------|------|
| 13:05-13:15 | 回到專案資料夾 | 跟做 | `cd Desktop/my-page`、`ls`、打開 `index.html` |
| 13:15-13:35 | 好指令 vs 壞指令 | 講 + 互動 | 壞：「幫我變好看」；好：「做成個人名片頁，低飽和色，三區塊」 |
| 13:35-14:00 | 改版任務 1 | 跟做 | 請 AI 增加版面：hero、技能、作品、聯絡 |
| 14:00-14:20 | 改版任務 2 | 跟做 | 請 AI 改風格：顏色、字體、間距、手機版 |
| 14:20-14:35 | 看懂 AI 做了什麼 | 講 + 跟做 | 用 `cat index.html` 看檔案；不用完全懂，只知道檔案被改了 |

**過關標準：** 學員能說出「我的網頁檔案叫 `index.html`，在 `my-page` 裡，可以請 AI 繼續改」。

---

### 休息 15 min（14:35 – 14:50）

---

### UNIT 4：保存版本 + 自由完成作品（14:50 – 16:05，75 min）

| 時間 | 主題 | 形式 | 細節 |
|------|------|------|------|
| 14:50-15:05 | 為什麼需要保存版本 | 講 | 用「改壞了想回上一版」帶入，不講協作平台 |
| 15:05-15:20 | Git 本機 demo | 講師 demo | `git init`、`git add .`、`git commit -m "first page"`；不 push |
| 15:20-15:45 | 自由完成作品 | 實作 | 三選一：個人頁、活動頁、店家介紹頁 |
| 15:45-15:58 | 本機展示準備 | 實作 | 確認瀏覽器能開；截圖；檔案命名正確 |
| 15:58-16:05 | 成果收斂 | 講 | 今天完成的是本機作品，不是公開網站 |

**Git 定位：** 只讓學員認識「版本保存」概念。學得快的人可以跟做本機 commit，但不列入過關標準。

**可刪減：** Git demo 可縮到 5 分鐘；自由完成作品不可刪。

|------|------|------|------|
| 16:15-16:25 | 作品展示 | 互動 | 2-3 位自願者展示本機頁面或截圖 |
| 16:25-16:35 | 今天真正學到什麼 | 講 | AI 指令、CLI 找檔案、本機 HTML、版本保存概念 |
| 16:35-16:43 | 下一步 | 講 | 回家改第二版；進階課再教 GitHub / Vercel / Supabase |

## 必教 / 可跳

| 類別 | 內容 | 原則 |
|------|------|------|
| 必教 | 打開終端機、`pwd`、`cd`、`ls`、`mkdir`、建立 `index.html`、本機開啟 HTML、請 AI 修改 | 沒有這些就做不出作品 |
| 可教 | `cat`、`cp`、`mv`、`rm` | 只在時間足夠時補 |
| Demo | Git 本機 commit | 建立版本保存概念 |
| 不教 | GitHub push、公開網址、Vercel、Supabase、環境變數、branch、conflict | 進階課內容 |

## 成果定義

### 必達成果

每位學員下課前要完成：

1. 有一個資料夾：`my-page`
2. 裡面有一個檔案：`index.html`
3. 可以用瀏覽器本機打開
4. 至少請 AI 修改過一次
5. 可以截圖或展示給旁邊的人看

### 不把這些當成果

- 有公開網址
- 有 GitHub repo
- 有部署到 Vercel
- 有接 Supabase
- 會完整 Git workflow

## 進階班銜接

單日班 = 本機 HTML 作品 + CLI 生存指令 + AI 修改能力。

進階班才進入：

- Git / GitHub 完整流程
- GitHub Pages / Vercel 部署
- Supabase 資料庫與後台
- 更完整的 AI coding workflow
- branch / conflict / PR 協作

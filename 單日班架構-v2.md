# AI 入門課 — 用 terminal 叫 AI 做出第一個本機 HTML（單日班架構 v2）

> 2026-06-18 改寫：課程從「CLI + Git + Agentic AI 工程工作流」改為「AI 入門體驗課」。terminal 是操作 AI 的入口，不是先學指令的地方。主工具是 opencode（本機 AI CLI），備援是 ChatGPT Codex 桌面版 / Claude Cowork / ChatGPT / Claude 網頁版。完全不教 Git、不公開網址、不部署。

## 設計變更摘要

| 項目 | 原設計 | 新設計 | 變更理由 |
|------|--------|--------|---------|
| 課程定位 | Agentic AI 入門，CLI 是門票、Git 是橋樑、AI 是目的地 | AI 入門體驗課，terminal 是操作 AI 的入口 | 學員多半只熟 ChatGPT 對話，先看到成果再補工具 |
| 主工具 | Claude Code / ChatGPT 網頁版 | opencode（本機 AI CLI） | opencode 免費、本機直接改檔，適合體驗「AI 直接動手」 |
| 備援工具 | 無明確備援 | ChatGPT Codex 桌面版、Claude Cowork、ChatGPT / Claude 網頁版 | opencode 裝不起來時不掉線 |
| 畢業成果 | CLI + Git + AI 一條龍，commit / push | 本機瀏覽器打開自己的 `index.html` | 降低失敗點，讓每個人感覺有做出東西 |
| Git | Unit 主線，學員跟做 add / commit / push | 完全不教 | 版控不是這堂的學習目標，移除以騰出時間做作品 |
| GitHub / 部署 | 學員推上 GitHub | 不教公開網址、不部署 | 避免帳號、權限、網路、部署平台拖垮初學者 |
| terminal 角色 | 先學一堆指令 | 叫 AI 做事，並看懂 AI 做了什麼；必要時才手動下 pwd / cd / ls / open 驗證 | 初學者先會用 AI，不先背指令表 |
| AI 段落 | 下午第二節才大量實作 | 午前就做出第一個 HTML，下午用 AI 改版 | 讓學員早點看到「我做到了」 |

## 基本資訊

- **時長：** 8 小時（09:00 – 17:00，含 30 min 安裝緩衝）
- **對象：** 一般民眾，無技術背景，頂多熟悉 ChatGPT 對話
- **定位：** AI 入門體驗課 — 用 terminal 作為操作 AI 的入口，做出第一個本機 HTML 作品
- **主工具：** opencode（本機 AI CLI）
- **備援工具：** ChatGPT Codex 桌面版、Claude Cowork、ChatGPT / Claude 網頁版
- **必達成果：** 每位學員都有一個本機可開啟的 `index.html`
- **不做事項：** 不教 Git、不公開網址、不教 Vercel / Supabase / GitHub Pages、不部署
- **平台：** Mac 和 Windows 同時教，投影片雙色標示
- **學員自備：** 筆電 + 充電器；現場統一帶安裝，預留 30 分鐘安裝緩衝

## 課程設計理念

### 一句話定位

> terminal 不是用來背指令的，是「叫 AI 做事，並看懂 AI 做了什麼」的入口。

學員今天學的不是工程師技能，而是：打開 terminal → 跟 opencode 說話 → AI 直接改檔 → 本機瀏覽器看到成果。必要時才讓學員手動下 `pwd` / `cd` / `ls` / `open` 等指令驗證檔案位置與內容。

### 三層能力架構

| 層次 | 能力 | 定位 | 學員帶走什麼 |
|------|------|------|-------------|
| AI 對話 | 把想法說清楚，讓 opencode 直接改檔 | 熟悉入口 | 會把「我要什麼網頁」講成 AI 能執行的指令，並看到 AI 自己改好檔案 |
| terminal 看懂 | 找檔案、進資料夾、開檔案、看 AI 做了什麼 | 操作入口 | 不怕黑畫面，知道作品在哪裡，能驗證 AI 改了什麼 |
| HTML 作品 | 本機網頁產出 | 可見成果 | 打開瀏覽器看到自己的頁面，能再請 AI 修改 |

### opencode 深度邊界

教到什麼程度，不教到什麼程度：

| 教 | 不教 |
|----|------|
| 基本對話（prompt 進、回答出） | MCP server 接接 |
| 直接改檔（AI 自己寫入 `index.html`） | hook 自動化 |
| 本機開啟（`open` / `start` 看成果） | context / memory 管理 |
| 內建 skill 概念（提一句「opencode 有內建 skill 可以做事更快」，不實作） | 自訂 skill 撰寫 |
| 切換 model / provider（只在裝不起來時提） | 多 agent 編排 |

### 設計原則

- **成果優先**：先讓學員看到自己的網頁，再補工具概念。
- **AI 動手，學員看懂**：opencode 直接改檔，學員負責看懂、驗證、再下指令，不手刻 HTML。
- **少教名詞**：每個手動指令都要服務 `index.html` 成果，不為了完整而教。
- **不教 Git**：完全不提版本控、commit、branch。作品就是一個資料夾裡的 `index.html`。
- **不部署**：公開網址、Vercel、Supabase、GitHub Pages 全部移到進階課。
- **每段都有可見回饋**：AI 改檔後看得到網頁變化，手動驗證後看得到檔案位置。

## 課程流程

### UNIT 0：環境安裝（08:30 – 09:00，30 min 緩衝）

| 時間 | 主題 | 形式 | 細節 |
|------|------|------|------|
| 08:30-08:45 | 報到 + 確認筆電 | 講 | 充電中、瀏覽器可開、可連網 |
| 08:45-09:00 | 安裝 opencode | 跟做 | 講師統一帶安裝；Mac / Windows 雙色標示 |

**安裝救援順序（opencode 裝不起來時）：**

1. opencode（主）
2. ChatGPT Codex 桌面版
3. Claude Cowork
4. ChatGPT / Claude 網頁版（最低門檻，手動貼 HTML 到檔案）

> 只要學員有任一工具能產生 HTML 並貼進 `index.html`，就算過關。安裝不是學習目標。

---

### UNIT 1：開場 + AI 能做什麼 + terminal 生存感（09:00 – 10:30，90 min）

| 時間 | 主題 | 形式 | 細節 |
|------|------|------|------|
| 09:00-09:08 | 歡迎 + 課程定位 | 講 | 今天是 AI 入門體驗課，不是工程師訓練；用 terminal 叫 AI 做出本機網頁 |
| 09:08-09:20 | 自我介紹 + 作品展示 | 講 | 用真實作品建立可信度，但不炫技 |
| 09:20-09:35 | 從 ChatGPT 對話到 AI 做事 | 講 + 互動 | 從學員熟悉的 ChatGPT 切入，展示「一句話變成網頁」；再展示 opencode 直接把檔案寫好 |
| 09:35-09:45 | 黑畫面恐懼 | 互動 | 問感受，建立今天 terminal 是「叫 AI 做事、看 AI 做了什麼」，不是背指令 |
| 09:45-10:00 | 打開終端機 + 看自己在哪 | 跟做 | Terminal / PowerShell、`pwd`、`ls`（手動驗證用） |
| 10:00-10:15 | 進資料夾 + 建資料夾 | 跟做 | `cd Desktop`、`mkdir my-page`、`cd my-page` |
| 10:15-10:30 | 建第一個檔案 | 跟做 | `touch index.html` / `New-Item index.html`，確認 `ls` 看得到 |

**過關標準：** 每個人終端機能進到 `my-page`，且 `ls` 看得到 `index.html`。

**terminal 角色說明：** 這段手動指令是為了讓學員「知道作品在哪裡」。之後主要靠 opencode 改檔，手動指令只在驗證時用。

**可刪減：** 自我介紹縮短；AI 展示從 4 張卡縮成 1 張「一句話變網頁」。

### UNIT 2：第一個本機 HTML — opencode 直接改檔（10:45 – 12:10，85 min）

| 時間 | 主題 | 形式 | 細節 |
|------|------|------|------|
| 10:45-11:00 | HTML 是什麼 | 講 | 不教語法大全，只講「網頁就是一個文字檔，瀏覽器讀它」 |
| 11:00-11:25 | opencode 基本對話 + 直接改檔 | 跟做 | 在 `my-page` 資料夾啟動 opencode；請它「做一個簡單個人介紹頁，寫進 index.html」；AI 自己寫入，學員不用手貼 |
| 11:25-11:40 | 本機瀏覽器打開 | 跟做 | Mac: `open index.html`；Windows: `start index.html`；看到自己第一個頁面 |
| 11:40-11:55 | 請 opencode 改第一版 | 跟做 | 「加標題、換顏色、加三個卡片、加聯絡方式」；AI 直接改檔，學員重新整理瀏覽器看變化 |
| 11:55-12:10 | 小成果確認 | 互動 | 每人瀏覽器看到自己的頁面；卡住者用講師範本救援 |

**過關標準：** 本機瀏覽器能打開 `index.html`，畫面不是空白，且能說出「這是 opencode 幫我寫的」。

**救援方案：** 準備 `starter-index.html`，opencode 完全裝不起來者切備援工具（ChatGPT Codex / 網頁版），手動貼 HTML 再修改名字。

**opencode 要點：** 強調「AI 自己改檔，你不用複製貼上」。這是 opencode 跟網頁版 ChatGPT 最大的差別，也是今天的主體驗。

### UNIT 3：AI 改版練習 — 從「能看」到「像作品」（13:05 – 14:35，90 min）

**可刪減：** 改版任務 2 的手機版部分，時間不足時跳過。

## 成果定義

### 必達成果

每位學員下課前要完成：

1. 有一個資料夾：`my-page`
2. 裡面有一個檔案：`index.html`
3. 可以用瀏覽器本機打開
4. 至少請 opencode（或備援工具）修改過一次
5. 可以截圖或展示給旁邊的人看

### 不把這些當成果

- 有公開網址
- 有 GitHub repo
- 有部署到 Vercel
- 有接 Supabase
- 會 Git workflow
- 會 opencode MCP / hook / 自訂 skill

Git 指令小抄

版控保命的常用指令 —— 程式碼再也不會真的搞丟
工作區
你改檔案的地方
git add
暫存區
準備好的改動
git commit
儲存庫
本地歷史存檔
git push
GitHub
雲端備份
每天都會用：日常循環
git status看現在什麼狀態（最常用，忘了就打）
git add 檔名把某個改動放進暫存區
git add .把所有改動一次放進暫存區
git commit -m "訊息"存成一個存檔點，附上說明
git push推上雲端 GitHub
git pull拉下別人最新的改動
git log看過去的存檔點歷史
git diff看這次到底改了哪些東西
第一次設定
git init在資料夾啟用版控
git clone 網址把雲端 repo 整個抓下來
git remote add origin 網址把本地連到 GitHub 上的 repo
git push -u origin main第一次推送（設好上游，之後只要 push）
分支：實驗不弄壞主線
git branch 名稱開一條新分支（實驗室）
git checkout 名稱切換到某條分支
git checkout -b 名稱開新分支並直接切過去
git merge 名稱把分支的成果合併回主線
後悔藥：救命四招
git checkout -- 檔名還沒 commit 改壞了 → 還原這個檔
git reset HEAD~1commit 錯了 → 回前一版（改動還留著）
git status不知道發生什麼事 → 先打它看狀態
git clone 網址整個資料夾炸了 → 重新拉一份乾淨的

commit 訊息小撇步：用「動詞 + 做了什麼」，例如 git commit -m "新增自我介紹頁"。訊息寫清楚，未來翻 git log 找版本時，現在的你會被未來的你感謝。

提示字元：Mac / Linux 是 $，Windows PowerShell 是 PS>，打指令時不用打它。
Git 指令在兩個平台寫法完全相同，放心照打。

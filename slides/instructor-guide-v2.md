## 教學提醒

### 上午 CLI（最容易出問題的時段）

- **terminal-check 是關鍵**：一定有人裝不好，預留 8-10 分鐘
- **Mac 用戶**：Terminal.app 在 Utilities 資料夾，用 Spotlight 最快
- **Windows 用戶**：建議裝 Windows Terminal，比內建 PowerShell 好用
- **rm 恐嚇時刻**：故意 `rm` 一個不重要檔案，問「能救回來嗎？」→ 不能
- **微休息要真的執行**：不要省略，90 秒就能恢復注意力

### 下午 Git（偏講解，節奏較可控）

- **為什麼要版控是重頭戲**：20 分鐘講解，讓學員真的理解動機
- **git init 跟做**：確認每個人都看到 `git log --oneline` 的輸出
- **push 到 GitHub**：如果 WiFi 太慢或有人沒帳號，講師 demo 即可
- **branch/衝突 = demo**：不需要學員操作，講師慢慢做，邊做邊解釋

### 下午 AI（最不可控）

- **Claude Code 安裝**：課前已預裝 Node.js，講師提前測好
- **跟做段是關鍵**：25 分鐘，助教必須全場巡，落後的人直接跟講師畫面抄
- **自由嘗試有硬上限**：13 分鐘到點就收，提醒截圖
- **AI 幻覺**：demo 中故意展示一個 AI 生成不存在指令的例子

### 冷場應對

| 狀況 | 解法 |
|------|------|
| 學員不動手 | 走到旁邊看螢幕：「來，我陪你打第一行」 |
| 做太快沒事做 | 「試試用不同的指令完成同一件事」 |
| 做太慢跟不上 | 「先跟著我的畫面看，截圖發群組」 |
| 全場安靜 | 正常 — 學生在打字就會安靜 |
| 問題太深 | 「這個很好，Q&A 時間再聊，先跟上進度」 |

### 時間緩衝

- 總可刪減時間 ~60min（佔全天 12.5%）
- 如果落後 15min 以內：砍環境變數口頭帶過
- 如果落後 30min：再砍 branch demo 改口頭
- 如果落後 45min：再砍衝突 demo + 自由嘗試
- 最壞情況：Unit 5 只做回顧 + Q&A 15min

## 微休息執行指南

每 25 分鐘插 90 秒微休息。具體做法：

1. 投影片出現「⚡ 微休息 90 秒」提示
2. 講師說：「站起來，伸個懶腰，看遠方 90 秒」
3. 計時器倒數（投影片上可視化）
4. 時間到：「好，我們繼續」

**不要省略微休息**。研究顯示 90 秒休息可維持 20.6% 正確率優勢。

## 紅綠便利貼機制

每位學員桌上放兩張便利貼：
- 🟢 綠色 = 我 OK，跟上了
- 🔴 紅色 = 我卡住了，需要幫助

使用方式：
1. 開場時發放，解釋用法
2. 每段實作開始時提醒：「卡住就翻紅色」
3. 助教看到紅色 → 主動過去幫忙
4. 不用舉手，不用開口，翻貼就好

## Codespaces 備案流程

如果學員本地環境壞掉：

1. 開瀏覽器 → github.com/codespaces
2. 點「New codespace」
3. 30 秒後得到一個雲端終端機（已裝好 Git + Node.js）
4. 跟著課程繼續操作

講師提前測試：確保 Codespaces 可用、網速足夠。

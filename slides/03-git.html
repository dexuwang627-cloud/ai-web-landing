TIME MACHINE

Git 是你的時間機器

每一次 commit = 一個存檔點，像遊戲的 save point。

$ git log --oneline
a3f2b1e 完成最終版
c7d9e4a 修改第三章
1b4a8f2 初稿 ← 隨時可以回到這裡

像 Google Docs 的版本記錄，但更強 —— 完整掌控、離線可用、記得每一次改動的歷程

HANDS-ON

動手做：git init → add → commit

Mac / Linux
$ git init
Initialized empty Git repository
$ echo "Hello Git" > hello.txt
$ git add hello.txt
$ git commit -m "第一版"
# 改第二行，再存一次
$ git add . && git commit -m "第二版"
Windows PowerShell
PS> git init
Initialized empty Git repository
PS> "Hello Git" > hello.txt
PS> git add hello.txt
PS> git commit -m "第一版"
# 改第二行，再存一次
PS> git add . ; git commit -m "第二版"

過關標準：git log 看到兩筆記錄就成功了

REMOTE

把程式碼送上雲端：git push + GitHub

GitHub = 程式碼的雲端備份 + 協作平台（全世界最大）。
為什麼要推上去？備份、跨裝置、分享、協作。

Mac / Linux
$ git remote add origin https://github.com/...
$ git push -u origin main
→ 重新整理 GitHub，看到你的檔案了
Windows PowerShell
PS> git remote add origin https://github.com/...
PS> git push -u origin main
→ 重新整理 GitHub，看到你的檔案了

第一次設定上游後，之後每次只要 git push

BRANCH

Branch：平行宇宙

主線（main）= 正式版；分支（branch）= 實驗室。
開 branch = 開一條平行時間線，改壞了不影響主線。

# 開一條實驗分支
$ git branch feature
$ git checkout feature # 切過去改東西
# 改壞了也沒關係，主線完好
$ git checkout main # 切回主線
$ git merge feature # 滿意了再合併回來

類比：同一份報告，一人改結論、一人改附錄，最後合併成一份

CONFLICT

衝突是什麼？Git 說「你來決定」

兩人改同一行 → Git 不知道聽誰的 → 衝突。
衝突不是壞事 = Git 把決定權交給你。

# 打開檔案，你會看到這些標記：
<<<<<<< HEAD
我寫的版本：標題用「快速上手」
=======
同事寫的版本：標題用「新手教學」
>>>>>>> feature
# 手動選一邊、刪掉標記 → 然後
$ git add . && git commit

看到 <<<<<<< 不要緊張 —— 就是選 A 或選 B 而已

CHEAT SHEET

Git 小抄：六條指令走天下

git init開始版控 git add <檔名>放進暫存區 git commit -m "訊息"存入歷史 git push推上雲端 git pull拉下最新（別人改的） git log查看歷史

加碼一條：git status

這條最常用 —— 忘記現在什麼狀態，就打它

SAFETY NET

Git 給你的五條命

1

改壞了

git checkout -- <檔名> 還原這個檔

2

commit 錯了

git reset HEAD~1 回前一版

3

整個資料夾炸了

git clone 重新拉一份乾淨的回來

4

不知道現在什麼狀態

git status 先看狀態

5

推上去發現有錯

改完再 push 一次就好 —— 歷史都還在

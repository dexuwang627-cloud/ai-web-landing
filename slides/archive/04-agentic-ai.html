PARADIGM

對話式 vs 代理式 AI

對話式

  • 你問一句、AI 答一句
  • ChatGPT 網頁版
  • 你要自己複製貼上
  • 像坐計程車說「左轉右轉」

代理式

  • 你給目標、AI 自己完成
  • 讀檔、寫檔、幫你改頁面
  • 直接操作你的專案資料夾
  • 像說地址，司機自己導航

今天的主角是能動手改檔案的 AI —— 你負責提出需求和確認成果

CLAUDE CODE

Claude Code 四大特色

CLI

終端機操作

就在剛才的 my-page 資料夾裡跑

HTML

直接改檔案

請它修改 index.html，你再用瀏覽器確認

一條指令啟動，不需要 IDE 或特殊編輯器

E2E

看得到結果

從文字需求到本機頁面，不是只給你程式碼片段

LIVE DEMO

現場 Demo：一句話改出網頁

$ cd my-page
$ claude "請把 index.html 改成個人介紹頁，單檔 HTML，不用外部套件"
# 觀察 Claude：讀 index.html → 修改內容 → 存檔
$ cat index.html # 看檔案有沒有內容
$ open index.html # Windows: start index.html

關鍵：AI 產出的東西，你一定要用瀏覽器看過才算完成

TRY IT · Step 1–2

跟做：回到 my-page + 啟動 Claude

# Step 1：進入早上建的資料夾
$ cd my-page
# Step 2：確認 index.html 在這裡
$ ls
index.html
# Step 3：啟動 Claude，給明確指令
$ claude "請把 index.html 改成我的個人介紹頁，保留單檔 HTML"

指令寫法技巧：說清楚「做什麼」+「長什麼樣」

TRY IT · Step 3

跟做：AI 產出 → 瀏覽器檢查

為什麼要打開看？AI 也會寫錯，你是最後守門員。

$ open index.html
瀏覽器看到標題、介紹、卡片
重新整理後確認新版有出現
不滿意就再請 AI 修改

看不懂程式沒關係：先確認畫面有沒有符合你的需求

TRY IT · Step 4–6

跟做：打開 → 修改 → 重新整理

# Step 4：本機打開
$ open index.html
# Step 5：請 AI 改一版
$ claude "請加上三張卡片，風格低飽和、乾淨"
# Step 6：回瀏覽器重新整理
$ open index.html

你完成了一個完整循環：AI 改 → 你看 → 不滿意再改

THE LOOP

AI 日常循環

你負責「看」和「決定」，AI 負責「做」。
瀏覽器驗收是關鍵——看過才算完成。

# 你以後每天都可以用的循環
$ claude "請修改 index.html..." # AI 幫你改
$ cat index.html # 確認檔案有內容
$ open index.html # 本機打開
$ 重新整理瀏覽器 # 你檢查成果

CLI 是入口，HTML 是成果，AI 是幫手

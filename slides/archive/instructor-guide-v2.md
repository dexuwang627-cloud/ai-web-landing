## Unit 1：開場 + AI 能做什麼 + CLI 生存感（09:00–10:30，90 min）

| # | 內容 | 時間 | 講解重點 | 互動 | 可跳 |
|---|------|------|---------|------|------|
| 1 | 歡迎 + 新定位 | 8 min | 今天不是工程師訓練，是做出第一個本機 HTML | 問「有誰只用過 ChatGPT？」 | |
| 2 | 自我介紹 + 作品展示 | 12 min | 用真實作品建立可信度，不炫技 | 展示成果即可 | 可縮到 5min |
| 3 | ChatGPT 對話 → AI 做事 | 15 min | 從熟悉的對話切入，展示一句話變網頁 | 讓學員說想做哪種頁面 | |
| 4 | 黑畫面恐懼 | 10 min | 今天只學幾個必要指令 | 舉手調查 | |
| 5 | 打開終端機 | 15 min | Terminal / PowerShell 能打字就過關 | 助教巡場 | |
| 6 | `pwd` / `ls` | 10 min | 知道自己在哪、看得到檔案 | 全場跟做 | |
| 7 | `cd` / `mkdir` | 15 min | 進桌面、建立 `my-page` | 全場跟做 | |
| 8 | 建 `index.html` | 5 min | 確認 `ls` 看得到檔案 | 翻綠色便利貼 | |

**過關標準**：學員在 `my-page` 裡，`ls` 看得到 `index.html`。

**講師提醒**：不要在這段講 Git、GitHub、部署。任何進階問題先收進「下午/進階課再講」。

## 教學提醒

### 絕對主線

- 每個人都要看到自己的 `index.html`。
- 每個人至少請 AI 改過一次。
- 每個人都要知道檔案在哪個資料夾。
- 不公開網址，不 push，不教部署。

### 最容易出問題的地方

| 狀況 | 解法 |
|------|------|
| 找不到終端機 | 直接走到旁邊幫他開，先不解釋太多 |
| 路徑迷路 | 回桌面重建 `my-page`，不要花太久救舊路徑 |
| HTML 空白 | 貼 `starter-index.html`，先讓瀏覽器有畫面 |
| AI 回覆太複雜 | 給固定 prompt 範本，要求只改 `index.html` |
| 學員問部署 | 回答「這是進階課，今天先確保本機作品完成」 |

### 時間緩衝

- 如果落後 15min：縮短自我介紹與作品展示。
- 如果落後 30min：Git demo 縮成 5min。
- 如果落後 45min：刪掉 `cat index.html` 說明，直接自由完成作品。
- 最壞情況：只保住 `index.html` 本機開啟 + AI 改一次。

## 固定 Prompt 範本

### 第一版 HTML

```text
請幫我產生一個單檔 index.html。
主題是「我的個人介紹頁」。
內容包含：我的名字、三個興趣、一段自我介紹、聯絡方式。
風格要簡潔、低飽和色、適合初學者。
請把 HTML、CSS 都放在同一個檔案，不要使用外部套件。
```

### 改版 Prompt

```text
請修改目前的 index.html。
幫我加上 hero 區塊、三張卡片、頁尾。
整體風格要更像作品集，但不要太花。
請保留單檔 HTML，不要拆 CSS 或 JS。
```

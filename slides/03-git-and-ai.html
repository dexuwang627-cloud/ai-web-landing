VERSION CONTROL

Git 是什麼？

Git 是一個時間機器，記錄你每次修改。
出錯了？回到之前的版本就好。

$ ls

WORKFLOW

Git 的三個區域

1

工作目錄

你正在編輯的檔案

2

暫存區

準備提交的變更

儲存庫

已保存的版本歷史

$ git add index.html # 工作目錄 → 暫存區
$ git commit -m "加入首頁" # 暫存區 → 儲存庫

add = 選你要提交的變更｜commit = 存一個版本

START

開始一個 Git 專案

從零開始
$ mkdir my-app && cd my-app
$ git init
Initialized empty Git repository
$ touch README.md
$ git add .
$ git commit -m "初始提交"
從 GitHub 複製
PS> git clone https://github.com/...
Cloning into 'my-app'...
PS> cd my-app
PS> git status
On branch main — nothing to commit
git init建立新的 Git 儲存庫 git clone <url>從遠端複製一個專案 git status看目前狀態（超常用）
TRY IT

動手：第一次 commit

# 1. 建新專案
$ mkdir my-git-practice && cd my-git-practice
# 2. 初始化 Git
$ git init
# 3. 建一個檔案
$ echo "# My First Repo" > README.md
# 4. 加入並提交
$ git add .
$ git commit -m "初次提交"
# 5. 確認歷史
$ git log --oneline

確認：git log --oneline 出現你的第一次 commit 就成功了。

COMMIT

add + commit：存一個版本

# 修改了 index.html
$ git status
Changes not staged for commit:
modified: index.html
# 選擇要提交的變更
$ git add index.html
# 存一個版本
$ git commit -m "修改首頁標題"
[main a3f2b1e] 修改首頁標題
1 file changed, 3 insertions(+), 1 deletion(-)
git add .全部加到暫存區 git add <file>只加特定檔案 git commit -m "訊息"提交並寫訊息 git log --oneline看版本歷史（簡短）
BRANCH

分支：平行時空

分支讓你在不影響主線的情況下開發新功能。
做完再合併回來。

# 建立新分支
$ git branch feature-login
$ git checkout feature-login
# 或一行搞定
$ git checkout -b feature-login
# 做完之後，合併回主線
$ git checkout main
$ git merge feature-login

分支 = 平行宇宙。合併 = 把平行宇宙收束回主線。

TRY IT

動手：分支與合併

# 1. 建分支並切換
$ git checkout -b add-intro
# 2. 改檔案
$ echo "這是我的介紹" >> README.md
$ git add . && git commit -m "加入介紹"
# 3. 回主線並合併
$ git checkout main
$ git merge add-intro
# 4. 確認合併成功
$ git log --oneline

挑戰：再建一個 feature-footer 分支，加一行後合併回 main。

REMOTE

推送到遠端：GitHub

本地儲存庫是你的。GitHub 是雲端備份 + 協作平台。

# 第一次推上去
$ git remote add origin https://github.com/you/repo.git
$ git push -u origin main
# 之後每次推
$ git push
# 拉別人的更新
$ git pull
git push把本地提交推到 GitHub git pull從 GitHub 拉最新更新 git remote -v看遠端設定
CHEAT SHEET

Git 指令總覽

git init建立新儲存庫 git clone <url>從遠端複製 git status看目前狀態 git add .加到暫存區 git commit -m "msg"提交版本 git log --oneline看版本歷史 git branch <name>建分支 git checkout -b <name>建分支並切換 git merge <branch>合併分支 git push推到遠端 git pull拉遠端更新 git diff看修改內容

記住這組：status → add → commit → push

PARADIGM SHIFT

對話式 vs 代理式

對話式 AI

  • 你問，AI 答
  • 你要自己複製貼上
  • 一步一步手動操作
  • AI 是顧問

代理式 AI

  • 你說目標，AI 執行
  • AI 直接改檔案、跑指令

  • AI 是助手

對話式 = 你打字問 ChatGPT
代理式 = 你說「幫我建一個網站」，AI 自己開終端機做

UNIT 4 · 目的地

AI 協作 — 今天的目的地

你已經學會 CLI 和 Git——它們是工具。
AI 讓工具自己動起來。

接下來，你會親手完成「跟 AI 一起寫 code → commit → push」

PARADIGM

對話式 vs 代理式 AI

對話式

  • 你問一句、AI 答一句
  • ChatGPT 網頁版
  • 你要自己複製貼上
  • 像坐計程車說「左轉右轉」

代理式

  • 你給目標、AI 自己完成
  • 讀檔、寫檔、跑指令、commit
  • 直接操作你的終端機和 Git
  • 像說地址，司機自己導航

今天的主角是代理式 —— 它直接動手，不只是給你建議

LIVE DEMO

現場 Demo：看 AI 自己動手

$ mkdir demo-project && cd demo-project
$ git init
$ claude "幫我建一個 Python 計算機 CLI 工具"
# 觀察 Claude：讀檔 → 寫檔 → 跑指令
$ git diff # 先看改了什麼
$ git add .
$ git commit -m "feat: calculator CLI"
$ git push

關鍵：AI 產出的東西，你一定要自己看過再 commit

TRY IT · Step 1–2

跟做：建專案 + 啟動 Claude

# Step 1：建資料夾並進入
$ mkdir my-project && cd my-project
# Step 2：啟用版控
$ git init
Initialized empty Git repository in .../my-project/.git/
# 啟動 Claude，給明確指令
$ claude "幫我建一個 intro.txt，內容是自我介紹，3-5 行"

指令寫法技巧：說清楚「做什麼」+「長什麼樣」

TRY IT · Step 3

跟做：AI 產出 → diff 檢查

為什麼要先看 diff？AI 也會寫錯，你是最後守門員。

$ git diff
+ Hello, 我是 ___
+ 今天參加了 CLI + Git + AI 課程
+ 正在學習用 AI 協作寫程式

diff 看不懂沒關係：找綠色（新增）和紅色（刪除）就好

TRY IT · Step 4–6

跟做：add → commit → push

# Step 4：加入暫存區
$ git add .
# Step 5：寫下你的提交訊息
$ git commit -m "feat: add intro.txt"
[main (root-commit) abc1234] feat: add intro.txt
1 file changed, 3 insertions(+)
# Step 6：推上遠端
$ git push origin main

你完成了一個完整循環：AI 寫 → 你看 → 你 commit → 你 push

THE LOOP

AI 日常循環

你負責「看」和「決定」，AI 負責「做」。
diff 是關鍵——不看就不 commit。

# 你以後每天都可以用的循環
$ claude "做什麼" # AI 幫你做
$ git diff # 你檢查產出
$ git add . # 你決定納入
$ git commit -m "..." # 你留下紀錄
$ git push # 你推上遠端

CLI 是手，Git 是記憶，AI 是夥伴

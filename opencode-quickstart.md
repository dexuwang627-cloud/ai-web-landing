# opencode 快速入門

> 給 AI 入門體驗課的講師備課用。opencode 是一支裝在本機的 AI CLI，你在終端機裡跟它對話，它能直接幫你建檔案、改檔案。這份講義只教「做出第一個本機 HTML」需要會的部分，不教 Git、不部署、不接 MCP。

---

## 1. opencode 是什麼

opencode 是一支跑在終端機（Terminal / PowerShell）裡的 AI 工具。跟 ChatGPT 網頁版的差別：

| | ChatGPT 網頁版 | opencode |
|---|---|---|
| 在哪用 | 瀏覽器 | 終端機黑畫面 |
| 能不能動你電腦的檔案 | 不能 | 能，直接建檔、改檔 |
| 你要怎麼用 | 打字問問題 | 打字問問題，它直接做 |

一句話：**你講，它做，做在你電腦上。** 學員今天要的不是「學終端機指令」，而是「叫 AI 做事，並看懂 AI 做了什麼」。opencode 就是那個讓 AI 能動手 的入口。

---

## 2. 啟動對話模式

裝好 opencode 後（現場統一帶安裝），做兩件事：

1. 先進到你要做事的資料夾：
   ```
   cd Desktop/my-page
   ```
2. 直接打：
   ```
   opencode
   ```

畫面會切到對話介面，下面有輸入框。這就是「對話模式（chat mode）」，跟 ChatGPT 一樣打字問問題就好。

操作重點：

- `Tab` 切換 Plan / Build 模式。Plan 模式只給建議、不動檔案；Build 模式才會真的改檔。初學者一律用 Build 模式即可，Plan 模式是給想先看計畫再動手的人。
- `@` 可以在訊息裡指向某個檔案，例如 `@index.html`，AI 會先讀那個檔案再回答。
- 想離開：按 `Ctrl + C` 或輸入 `/quit`。

非互動用法（進階，學員不需要）：`opencode run "把標題改成藍色"` 會跑完直接結束，適合腳本化；課堂上用不到。

---

## 3. 叫 AI 建資料夾與檔案

先讓學員手動進到桌面、建好專案資料夾，確認「作品在哪裡」：

```
cd Desktop
mkdir my-page
cd my-page
```

進到 `my-page` 後啟動 opencode，直接用自然語言叫 AI 建檔：

> 幫我在這個資料夾建一個 index.html 空檔案。

opencode 會在終端機顯示它執行了什麼（例如 `touch index.html`），檔案就出現了。這時讓學員手動驗證一次：

```
ls
```

看到 `index.html` 就過關。

教學重點：不是要學員背 `touch`、`mkdir`，而是讓他們看到「我叫 AI 做，它真的做了，而且我看得到它做了什麼」。AI 做完會把用到的指令印出來，那就是現成的生存指令教材。

---

## 4. 請 AI 產生第一版 index.html

檔案建好後，直接在 opencode 對話框裡下指令。好指令的公式：**主題 + 風格 + 內容區塊**。

範例（個人名片頁）：

> 幫我在 index.html 做一個個人名片頁：標題是我的名字，下面放一段自我介紹、三個技能卡片、還有聯絡方式。配色用低飽和色，字體大一點方便閱讀。

範例（商家介紹頁）：

> 幫我做一個咖啡店介紹頁 index.html：上面是店名與 hero 圖區、中間放三樣招牌商品、下面放地址與營業時間。配色走暖色系。

範例（活動頁）：

> 幫我做一個社區市集活動頁 index.html：活動名稱當大標、日期地點、活動流程時間表、報名方式。風格活潑但不要太花。

opencode 會直接把內容寫進 `index.html`，不用自己複製貼上。這是它跟網頁版 AI 最大的差別，也是今天的主秀。

如果 AI 改錯了或寫壞了，打 `/undo` 可以回上一版；`/redo` 再做回來。可以連按多次。

---

## 5. 本機開啟預覽

opencode 只負責改檔案，預覽要用瀏覽器開。Mac 與 Windows 指令不同，投影片雙色標示：

| 系統 | 指令 |
|---|---|
| Mac | `open index.html` |
| Windows | `start index.html` |

打完後預設瀏覽器會開啟你的頁面。看到畫面不是空白，就過關。

之後每次請 AI 改完，存檔後回到瀏覽器按重新整理（Mac `Cmd + R`、Windows `Ctrl + R`）就能看到新版本。不用重開檔案。

注意：這是本機預覽，沒有網址、沒有公開，只有你自己電腦看得到。這是今天刻意不部署，避免帳號、權限、網路卡住初學者。

---

## 6. 看懂 AI 改了什麼

opencode 改完檔案後，終端機會顯示它動了哪些檔案、大致做了什麼。學員不需要懂 HTML 語法，但要建立三個習慣：

1. **看終端機的摘要**：opencode 會列出它改了哪個檔案、做了什麼動作。看完心裡有底再回瀏覽器看畫面。
2. **用 `@` 指向檔案問 AI**：例如 `@index.html 這裡面的標題是什麼？`，AI 會讀檔案後回答。這是讓學員「不用自己看懂程式碼也能掌握內容」的方法。
3. **必要時手動看檔案內容**：`cat index.html` 會把檔案內容印在終端機。不用懂每一行，只要知道「這就是我的網頁，它被改過了」。

修改循環：

```
下指令 → opencode 改檔 → 看終端機摘要 → 瀏覽器重新整理 → 看畫面 → 再下指令
```

這個循環跑順了，學員就具備「回家自己繼續改」的能力，也是今天的核心成果。

常見修改指令範例：

> 把標題顏色改成深藍色，字再大一點。
> 在聯絡方式下面加一個 footer，寫上版權年份。
> 把三個技能卡片排成一行，手機上看要變成直排。
> 整體配色換成莫蘭迪色系，背景用淺米色。

## 7. 內建 skill 使用範例

opencode 有內建 skill，可以把它想成「AI 預裝的小工具」。初學者不需要自己設定，只要會叫 AI 用就好。這堂課只示範、不實作 MCP / hook / context 等進階設定。

### /init — 讓 AI 認識你的專案

在 `my-page` 裡啟動 opencode 後，輸入：

> /init

opencode 會掃描這個資料夾，產生一份 `AGENTS.md`，內容是它對這個專案的理解（例如「這是一個個人名片 HTML 專案，主檔案是 index.html」）。之後每次對話，AI 都會帶著這份理解，回答更準。

學員這堂課專案很小，`/init` 效果有限，但讓他們知道「AI 可以記住專案脈絡」這個概念即可。

### /undo、/redo — 改壞了救命用

前面提過，這裡強調使用時機：

> 改壞了、畫面跑掉了、不喜歡這次的結果 → 先打 `/undo` 回上一版，再重新下指令。

這是初學者最需要的「安全網」。告訴學員：不怕改壞，因為可以 undo。

### /share — 分享對話給講師

卡住時可以打：

> /share

opencode 會產生一個對話分享連結。學員把連結貼到 LINE 群組，講師就能看到他跟 AI 講了什麼、AI 做了什麼，直接診斷卡在哪。這比截圖黑畫面更有效。

### 不在這堂課的範圍

以下屬於進階，今天只點到名詞，不實作：

- MCP server（接外部工具）
- hook（自動化觸發）
- custom agent（自訂 AI 角色）
- LSP（程式語言伺服器）

學員只要記得：opencode 還能做更多，回家有興趣再摸索。進階課才會正式教。

## 8. 講師備課速查

學員今天要會的 opencode 動作只有五個：

1. `opencode` 啟動對話模式
2. 用自然語言叫 AI 建檔、改檔
3. `/undo` / `/redo` 回版
4. `/init` 讓 AI 記住專案
5. `/share` 求助

搭配的本機指令（只在驗證時手動下）：

| 指令 | 用途 | Mac / Windows |
|---|---|---|
| `pwd` | 看自己在哪 | 共用 |
| `cd` | 進資料夾 | 共用 |
| `ls` / `dir` | 看檔案清單 | Mac `ls`、Windows `dir` |
| `mkdir` | 建資料夾 | 共用 |
| `open` / `start` | 開瀏覽器預覽 | Mac `open`、Windows `start` |
| `cat` | 看檔案內容 | Mac 原生、Windows 用 `type` |

過關標準：學員能說出「我的網頁檔案叫 `index.html`，在 `my-page` 裡，可以用 opencode 請 AI 繼續改，瀏覽器重新整理就看得到」。
